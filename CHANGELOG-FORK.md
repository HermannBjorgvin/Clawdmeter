# Отличия этого форка от upstream

Форк [HermannBjorgvin/Clawdmeter](https://github.com/HermannBjorgvin/Clawdmeter):
устройство превращено из пассивного индикатора лимитов в настольный «пейджер»
Claude Code — со звуковыми оповещениями о событиях сессий, календарными
напоминаниями и русским интерфейсом. Проверено на Waveshare
ESP32-S3-Touch-AMOLED-2.16 + macOS-демоне.

*In English: this fork turns the device into a desk-side Claude Code pager —
typed sound/screen alerts driven by Claude Code hooks (waiting for input /
permission / done), meeting reminders from an ICS feed or the macOS system
calendar, a live activity indicator with per-session-count animations, a
third model-scoped limit gauge, and a runtime-switchable RU/EN interface
(`lang = ru|en` in the daemon config; defaults to English).*

## Локализация

- Все тексты прошивки вынесены в таблицу строк (`firmware/src/lang.{h,cpp}`)
  с русским и английским наборами, включая ~90 шуточных статусов
  («Клодит…», «Комбобулирует…»).
- Язык переключается параметром `lang = ru|en` в конфиге демона
  (`~/.config/claude-usage-monitor/config`), выбор сохраняется на устройстве
  (NVS) — экран сопряжения говорит на нужном языке уже при следующей
  загрузке. Без параметра поведение как в upstream (английский).
- Кириллица в фирменных шрифтах через LVGL-fallback: Golos Text → Styrene,
  PT Serif → Tiempos; моноширинный перегенерирован из DejaVu Sans Mono.

## Оповещения от Claude Code

- Хуки Claude Code (`Notification` / `Stop` / `UserPromptSubmit` /
  `PostToolUse` → `tools/claude-attention-hook.sh`) классифицируют события,
  демон пересылает их на устройство, прошивка показывает типизированный
  экран со своей мелодией (синтез синусов через ES8311, без PCM-клипов):
  - **«Клод ждёт ответа»** — сессия ждёт ввода;
  - **«Нужно разрешение»** — запрос разрешения на инструмент;
  - **«Готово!»** — ход завершён (ходы короче 45 с не сигналят);
  - **«Лимит близко!»** — пересечение 80 % / 95 % сессионного окна;
  - **«Лимиты обновились!»** — сброс сессионного окна (подмигивание).
- В шапке экрана оповещения — имя проекта (basename git-корня), до двух
  строк с переносом. Начали печатать на маке — экран гаснет сам.
- Ложные срабатывания отфильтрованы: фоновые задачи и параллельные агенты
  проверяются честно (lsof + свежесть транскриптов), системные уведомления
  не считаются вводом пользователя, автономные сессии не будят.

## Календарь

- Напоминания о встречах **за 15 и за 5 минут** (настраивается,
  `cal_remind`) и отдельное событие **«Встреча началась!»** (жёлтым,
  в двухминутном окне после начала — опоздавший демон молчит).
- Источник — параметр `cal_ics_url`: https-ссылка на ICS (опубликованный
  календарь Outlook/Google), локальный `.ics` или `eventkit` — системный
  календарь macOS через подписанный Swift-хелпер `daemon/calnext`
  (голому python macOS отказывает в EventKit без диалога; сборка —
  `daemon/build_calnext.sh`).
- ICS-путь разворачивает повторяющиеся события (RRULE, виндовые TZID),
  отбрасывает all-day и отменённые; срабатывает только ближайший активный
  порог, пометка «отправлено» — после успешной BLE-записи.

## Лимиты

- Данные берутся с read-only `GET /api/oauth/usage` (не тратит лимит);
  probe-запрос upstream оставлен как fallback и для Enterprise.
- **Третий индикатор** — модельный недельный лимит (`weekly_scoped`,
  например Fable): компактный чип в панели «Неделя» с окраской по порогам.
- Рядом со временем «через сколько сброс» — время сброса на часах:
  «Сброс через 4ч 20м (вс 00:20)».
- При ошибках опроса устройство честно показывает причину («Обновите
  токен», «Нет сети»…) вместо устаревших цифр; интервал опроса растёт
  экспоненциально. Токен сознательно не рефрешится демоном — ротация
  refresh-токена может разлогинить Claude Code.

## Живой индикатор

- Счётчик активных сессий (heartbeat-хуки) — «·N» в углу и статус
  «Отдыхает» с застывшей звёздочкой, когда Клод ничем не занят.
- Анимации заставки зависят от числа работающих сессий: 1–2 — рабочие,
  3 — «dance sway dj», 4 — «dance bounce dj», 5+ — «dance djmix» в двойном
  темпе.

## Прочее

- Процент батареи числом рядом с иконкой, виден и при зарядке.
- Гашение экрана через 5 минут бездействия Клода (авто-пробуждение при
  активности и событиях).
- Серийные команды `usage` / `splash` — переключение экранов без кнопок
  (для QA-скриншотов через `screenshot.sh`).
- Исправление upstream-бага: `doc["x"] | false` в ArduinoJson молча
  отбрасывает int-флаги — заменено на `.as<bool>()`.
