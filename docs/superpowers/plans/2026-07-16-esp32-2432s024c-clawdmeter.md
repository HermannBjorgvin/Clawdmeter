# ESP32-2432S024C Clawdmeter Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build, flash, and validate Clawdmeter on Gustavo's USB-powered ESP32-2432S024C, then install and verify the native Windows BLE daemon.

**Architecture:** Add one board-specific HAL folder and PlatformIO environment while keeping Clawdmeter's shared firmware intact. Extract only pure touch, power-button, and numeric UI layout helpers so behavior can be tested before the corresponding embedded integration is written.

**Tech Stack:** PlatformIO Core, pioarduino ESP32 Arduino core, C++17, Adafruit ST7789/GFX, LVGL 9, NimBLE-Arduino, ArduinoJson, Unity, pytest, Python 3.13, Bleak/WinRT.

## Global Constraints

- Target exactly the verified ESP32-D0WD-V3 revision 3.1 with 4 MB flash and no PSRAM on COM3.
- Use ST7789 SPI pins SCLK 14, MOSI 13, MISO 12, CS 15, DC 2, reset absent, and backlight GPIO 27.
- Use CST816/CST820-compatible I2C touch at address 0x15 with SDA 33, SCL 32, and reset 25.
- Keep the display portrait at 240 x 320 and preserve both upstream splash and usage screens.
- Keep the display awake while USB-powered; no battery, PMU, IMU, sound, rotation, or HID typing buttons.
- Use GPIO 0 as the PWR-style short/long/release control without sending Space.
- Keep upstream Windows daemon behavior and 60-second polling.
- Do not add Wi-Fi, OTA, SD, speaker, RGB effects, Codex usage, or unrelated refactors.
- Do not claim hardware success without build, upload, serial, visual, touch, BLE, and live-payload evidence.

```cpp
#pragma once
#include <stdint.h>

struct TouchPoint { uint16_t x; uint16_t y; };

inline TouchPoint map_touch_to_portrait(uint16_t raw_x, uint16_t raw_y) {
    return {
        raw_x > 239 ? 239 : raw_x,
        raw_y > 319 ? 319 : raw_y,
    };
}
```

- [ ] **Step 5: Implement the validated CST816/CST820 reader**

uses 400 kHz I2C, reads six bytes from register 0x01 at address 0x15, derives
`pressed`, raw X, and raw Y, then calls `map_touch_to_portrait`. A failed I2C
transaction returns `pressed=false` without delay or retry loops.
