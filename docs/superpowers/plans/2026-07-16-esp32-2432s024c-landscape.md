# ESP32-2432S024C Landscape Build Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a separately selectable 320x240 landscape firmware for the ESP32-2432S024C with USB on the left, while preserving the portrait build and correcting provider freshness/unavailability defects found in final review.

**Architecture:** A new PlatformIO environment adds `BOARD_LANDSCAPE` to the existing board port. Board-local display and touch code converts the native 240x320 panel into a logical 320x240 canvas, while shared layout metrics select a horizontal two-card profile. The v2 payload gains explicit local-provider tombstones, and the parser update mask is passed into the UI so only providers actually present in a payload change validity or freshness.

**Tech Stack:** PlatformIO, Arduino ESP32, Adafruit ST7789, CST816/CST820 touch over I2C, LVGL 9.2, ArduinoJson 7, Python 3, pytest, Unity.

## Global Constraints

- Preserve `esp32_2432s024c` as the 240x320 portrait environment.
- Add `esp32_2432s024c_landscape` as the 320x240 environment with USB physically on the left.
- Keep USB serial transport, four pages, page order, 12-second automatic interval, and 30-second post-touch holdoff.
- Keep the robot orange and unmirrored; do not add a battery indicator.
- Native ST7789 initialization remains 240x320 in both builds.
- Serial command buffer remains 768 bytes; compact-payload tests must stay below it.
- Omitted provider means not updated; explicit `null` means attempted but unavailable.
- Never serialize prompt, response, filesystem path, session name, or task title.
- Use `pio run ... -j 1` for final builds because parallel LVGL compilation has produced an intermittent GCC ICE on this machine.
- Do not push the branch without Gustavo's explicit confirmation.
